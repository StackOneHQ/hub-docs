## Generating OAuth Credentials

<Steps>
  <Step title="Add OAuth Application">
    Under _Company Setup_ On the `Login Config` tab, find `OAuth Applications` and click `Add OAuth
        Application`.

    <Frame>
      <img
        className="rounded-md"
        style={{ margin:"0 auto",border:"1px solid #efefef" }}
        alt="login config"
        src="/images/ukgready/image12.png"
      />
    </Frame>
    <Frame>
      <img
        className="rounded-md"
        style={{ margin:"0 auto",border:"1px solid #efefef" }}
        alt="OAuth Application"
        src="/images/ukgready/image13.png"
      />
    </Frame>
  </Step>
  <Step title="Generate OAuth">
    If the integration is configured with an Authentication Type of `OAuth`:
    - Select `Application Type` to be `Interactive`
    - Add the following Redirect URL: `https://api.stackone.com/connect/oauth2/ukgready/callback`

    If not, please proceed to the next step.

    <Frame>
      <img
        className="rounded-md"
        style={{ margin:"0 auto",border:"1px solid #efefef" }}
        alt="login config"
        src="/images/ukgready/image14.png"
      />
    </Frame>
    Once you're happy with the configuration - click the `Save` button and enter
    the `Client_ID` in the respective fields during the account linking process.
  </Step>
  <Step title="Generate OAuth Client Credentials Grant">
    If the authorisation type is `OAuth Client Credentials Grant` then select
    `Application Type` to be `Machine to Machine`.

    Generate the `Client Secret` and select the appropriate access control and security profiles.

    The selected Company Security Profile requires permissions for retrieving the data to be requested with this connection. 

    <Frame>
      <img
        className="rounded-md"
        style={{ margin:"0 auto",border:"1px solid #efefef" }}
        alt="login config"
        src="/images/ukgready/image15.png"
      />
    </Frame>
    Once you're happy with the configuration - click `Save` and enter
    the `Client_ID` and `Client Secret` in the respective fields during the account linking process.
  </Step>
</Steps>

## Job Change Reason Codes

UKG Ready requires a Job Change Reason Code when assigning a job to a new hire, or changing an employee's job. At least one of these fields is required if employee jobs will be assiged via this connection.

<Steps>
  <Step>
    Click ≡ at the top-left corner.

    <Frame>
      <img
        className="rounded-md"
        style={{ margin:"0 auto",border:"1px solid #efefef" }}
        alt="Company Setup"
        src="/images/ukgready/image1.png"
      />
    </Frame>
  </Step>
  <Step>
    Navigate to `Settings > HR Setup > Company Setup > Job Change Reason Codes`.

    <Frame>
      <img
        className="rounded-md"
        style={{ margin:"0 auto",border:"1px solid #efefef" }}
        alt="global setup"
        src="/images/ukgready/image16.png"
      />
    </Frame>
  </Step>
  <Step>
    The existing codes for your company will be displayed here.

    For the **Job Change Reason Code - New Hire** field, we recommend using a code of Type "New Hire".

    For the **Job Change Reason Code - Other** field, we recommend using a code of Type "Other".

    You can create a new code by clicking the `ADD NEW` button in the top-right.

    Copy the **Name** of each code you wish to use.
  </Step>
</Steps>

## Connecting with StackOne

Upon reaching the Link Account page, enter the following credentials:

- UKG Ready URL
- Client ID
- Client Secret
- Company Short Name
- Job Change Reason Code - New Hire (optional)
- Job Change Reason Code - Other (optional)

Proceed by clicking the Connect button.
