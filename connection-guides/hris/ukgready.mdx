If you've been directed to StackOne to integrate with UKG Ready, the following steps will help you understand the process and any necessary actions to configure a successful integration.

## Generating OAuth Credentials

<Steps>
  <Step title="Add OAuth Application">
    Under *Company Setup* in the `Login Config` tab, locate `OAuth Applications` and click `Add OAuth
    Application`.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Login Config"
        src="/images/ukgready/image12.png"
      />
    </Frame>
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="OAuth Application"
        src="/images/ukgready/image13.png"
      />
    </Frame>
  </Step>
  
  <Step title="Generate OAuth">
    If the authorization type is set to `OAuth`, select `Interactive` as the `Application Type`. If not, proceed to the next step.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Login Config"
        src="/images/ukgready/image14.png"
      />
    </Frame>
    Once you've confirmed the configuration, click the `Save` button and enter
    the `Client_ID` in the respective fields during the account linking process.
  </Step>
  <Step title="Generate OAuth Client Credentials Grant">
    If the authorization type is `OAuth Client Credentials Grant`, select
    `Machine to Machine` as the `Application Type`.

    Generate the `Client Secret` and select the appropriate access control and security profiles.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Login Config"
        src="/images/ukgready/image15.png"
      />
    </Frame>
    After confirming the configuration, click `Save` and enter
    the `Client_ID` and `Client Secret` in the respective fields during the account linking process.

  </Step>
</Steps>

## Job Change Reason Codes

UKG Ready requires a Job Change Reason Code when assigning a job to a new hire or changing an employee's job. At least one of these fields is required if employee jobs will be assigned via this connection.

<Steps>
  <Step>
    Click the ≡ menu icon at the top-left corner.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Company Setup"
        src="/images/ukgready/image1.png"
      />
    </Frame>
  </Step>

  <Step>
    Navigate to `Settings > HR Setup > Company Setup > Job Change Reason Codes`.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Global Setup"
        src="/images/ukgready/image16.png"
      />
    </Frame>
  </Step>

  <Step>
    The existing codes for your company will be displayed here. 

    For the **Job Change Reason Code - New Hire** field, we recommend using a code of Type "New Hire".

    For the **Job Change Reason Code - Other** field, we recommend using a code of Type "Other".

    To create a new code, click the `ADD NEW` button in the top-right corner.

    Copy the **Name** of each code you wish to use.
  </Step>
</Steps>

## Connecting with StackOne

When you reach the Link Account page, enter the following credentials:
- UKG Ready URL
- Client ID
- Client Secret
- Company Short Name
- Job Change Reason Code - New Hire (optional)
- Job Change Reason Code - Other (optional)

Click the Connect button to proceed.
