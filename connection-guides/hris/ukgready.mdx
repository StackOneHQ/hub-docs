To integrate UKG Ready with StackOne, follow these steps to gather the necessary credentials and configure your account settings.

<Steps>
  <Step title="Login to UKG Ready Dashboard">
    Sign in to your UKG Ready account.
    
    Copy your **UKG Ready URL** from the address bar (it should end with `.rdy.ukg.dev`). Save this URL as you'll need it later.

    </Frame>
  </Step>

<Step title="Access Company Setup">
  Click the menu icon (≡) in the top-left corner.
  <Frame>

      className="rounded-md"

      src="/images/ukgready/image1.png"
    />
  </Frame>
  Navigate through: `Settings > Global Setup > Company Setup`
  <Frame>
    <img

      alt="Global Setup"
      src="/images/ukgready/image2.png"

  </Frame>
  <Frame>

      className="rounded-md"

## Generating OAuth Credentials

<Steps>
  <Step title="Create OAuth Application">
    In the `Login Config` tab under *Company Setup*, locate `OAuth Applications` and select `Add OAuth
    Application`.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Login Config"
        src="/images/ukgready/image12.png"
      />
    </Frame>
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Add OAuth Application"
        src="/images/ukgready/image13.png"
      />
    </Frame>
  </Step>
  
  <Step title="Configure OAuth Settings">
    For authorization type `OAuth`, select `Interactive` as the `Application Type`.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="OAuth Configuration"
        src="/images/ukgready/image14.png"
      />
    </Frame>
    Click `Save` and copy the generated `Client_ID`. You'll need this during the
    account linking process.
  </Step>

  <Step title="Configure OAuth Client Credentials">
    For authorization type `OAuth Client Credentials Grant`:
    1. Select `Machine to Machine` as the `Application Type`
    2. Generate a `Client Secret`
    3. Configure appropriate access control and security profiles
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Client Credentials Configuration"
        src="/images/ukgready/image15.png"
      />
    </Frame>
    Click `Save` and copy both the `Client_ID` and `Client Secret`. You'll need
    these during the account linking process.
  </Step>
</Steps>

## Job Change Reason Codes

UKG Ready requires Job Change Reason Codes when assigning jobs to new hires or modifying existing employee positions. At least one code is required if you plan to manage employee jobs through this integration.

<Steps>
  <Step title="Access Menu">
    Click the menu icon (≡) in the top-left corner.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Company Setup"
        src="/images/ukgready/image1.png"
      />
    </Frame>
  </Step>

  <Step title="Navigate to Job Change Codes">
    Go to `Settings > HR Setup > Company Setup > Job Change Reason Codes`.
    <Frame>
      <img
        className="rounded-md"
        style={{ margin: "0 auto", border: "1px solid #efefef" }}
        alt="Global Setup"
        src="/images/ukgready/image16.png"
      />
    </Frame>
  </Step>

  <Step title="Select or Create Codes">
    Review your company's existing codes. You'll need:

    - For **Job Change Reason Code - New Hire**: Select a code with Type "New Hire"
    - For **Job Change Reason Code - Other**: Select a code with Type "Other"

    To create a new code, click the `ADD NEW` button in the top-right corner.

    Make note of the **Name** for each code you plan to use.
  </Step>
</Steps>

## Connecting with StackOne

On the Link Account page, provide these credentials:
- UKG Ready URL
- Client ID
- Client Secret
- Company Short Name
- Job Change Reason Code - New Hire (optional)
- Job Change Reason Code - Other (optional)

Click Connect to complete the integration.
