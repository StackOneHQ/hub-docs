If you've been directed to StackOne to integrate with HiBob, follow these steps to configure a successful integration.

## Creating a New Service User

<Steps>
  <Step title="Navigate to Service Users">
    1. Using the left menu, select Settings > Integrations
    2. Select Automation under Categories
    3. Click Manage under the Service Users icon

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Service Users" src="/images/hibob-service-user/image1.png" />
    </Frame>
  </Step>

  <Step title="Create New Service User">
    Click on the "New Service User" button.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="New Service User" src="/images/hibob-service-user/image2.png" />
    </Frame>
  </Step>

  <Step title="Fill in Details">
    Enter all required information in the form and click Next.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Fill Details" src="/images/hibob-service-user/image3.png" />
    </Frame>
  </Step>

  <Step title="Copy Service User ID and Token">
    Save both the Service User ID and Token in a secure location. 

    <Warning>
      The token cannot be retrieved after this step. However, you can reset it to generate a new one if needed.
    </Warning>

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Service User ID and Token" src="/images/hibob-service-user/image4.png" />
    </Frame>
  </Step>

  <Step title="Finish Creation">
    Click Done to complete the Service User creation process.
  </Step>
</Steps>

## Access Permissions for the Service User

The newly created Service User account requires specific access permissions to retrieve data. You'll need to either create a new Permissions Group or use an existing one. For detailed instructions on creating custom permission groups, refer to HiBob's documentation:

[Creating a Custom Permissions Group](https://help.hibob.com/hc/en-us/articles/4409776408209-Create-a-custom-permission-group)

## Assigning Permissions to the Service User

<Steps>
  <Step title="Navigate to Permissions Groups">
    1. Go to Settings in the left menu
    2. Click Permissions Groups under the Account heading
    3. Select the group created for your Service Users

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Permissions Groups" src="/images/hibob-service-user/image5.png" />
    </Frame>
  </Step>

  <Step title="Edit Permissions Group">
    1. Click the Group Actions dropdown
    2. Select Edit Details from the list

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Edit Details" src="/images/hibob-service-user/image6.png" />
    </Frame>
  </Step>

  <Step title="Select People By Condition">
    For new groups, select "Select People By Condition." If this option is already selected, proceed to the next step.

    <Warning>
      Service Users cannot be added to groups using the "Select People by Name" option. Create a new permissions group if needed.
    </Warning>

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Select People By Condition" src="/images/hibob-service-user/image7.png" />
    </Frame>
  </Step>

  <Step title="Add Service Users">
    1. In the "Select People By Condition" modal, click the dropdown under "Add Specific Employees"
    2. Select either all Service Users or individual ones as needed
    3. Click Apply twice
    4. Click Save to confirm changes

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Add Specific Employees" src="/images/hibob-service-user/image8.png" />
    </Frame>
  </Step>
</Steps>
