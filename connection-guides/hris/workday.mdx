## Finding your Workday Tenant

<Steps>
  <Step title="Log into Workday">
    Log into your Workday account. In the address bar at the top of your browser window, locate your tenant name. It appears immediately after `workday.com/` in the URL.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Workday Tenant" src="/images/workday/image1.png" />
    </Frame>
  </Step>
</Steps>

## Finding the Web Services Endpoint

<Steps>
  <Step title="Navigate to Public Web Services">
    Access the `Public Web Services` report.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Human Resources Web Service" src="/images/workday/image2.png" />
    </Frame>
  </Step>

  <Step title="Locate Human Resources">
    Find the Human Resources section and hover over it to interact with the menu. Click the three-dots menu, select `Web Service`, then click `View SDL` (Note: The page may take a moment to load completely).

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="WSDL Service" src="/images/workday/image3.png" />
    </Frame>
  </Step>

  <Step title="Locate the WSDL Service">
    Either search for `wsdl:service` in the file or scroll to the bottom of the page. You should see something similar to this:

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="WSDL Service" src="/images/workday/image4.png" />
    </Frame>

    Copy everything before `service` in the location attribute, excluding the `https://` prefix. For example, in the tenant shown above, this would be `wd2-impl-services1.workday.com/ccx`. Your tenant may have a different format (e.g., `wd5-services1.myworkday.com/ccx`).
  </Step>
</Steps>

## Set up an Integration System User

<Steps>
  <Step title="Search for Create Integration System User">
    Log in to your Workday tenant through the Workday portal. In the Search field, enter "Create Integration System User".

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Create Integration System User" src="/images/workday/image5.png" />
    </Frame>
  </Step>

  <Step title="Select the Task">
    Click on the "Create Integration System User" task.
  </Step>

  <Step title="Configure Account Information">
    In the Account Information section of the "Create Integration System User" page, enter your desired username and password.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Account Information" src="/images/workday/image6.png" />
    </Frame>
  </Step>

  <Step title="Complete the Process">
    Click the OK button to create the Integration System User.
  </Step>
</Steps>

## Add the Integration System User to a Security Group

<Steps>
  <Step title="Access Security Group Creation">
    In the Search field, enter "Create Security Group" and select the "Create Security Group" task.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Create Security Group" src="/images/workday/image7.png" />
    </Frame>
  </Step>

  <Step title="Configure Security Group Settings">
    On the "Create Security Group" page:
    1. From the Type of Tenanted Security Group dropdown menu, select "Integration System Security Group"
    2. Enter a descriptive name in the Name field

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Security Group Type" src="/images/workday/image8.png" />
    </Frame>
  </Step>

  <Step title="Save Security Group">
    Click the OK button to create the Security Group.
  </Step>

  <Step title="Complete Security Group Configuration">
    On the "Edit Integration System Security Group (Unconstrained)" page, enter the same name you used when creating the Integration System User in the previous section. Click OK to save your changes.
  </Step>
</Steps>

## Configure Domain Security Policy Permissions

<Steps>
  <Step title="Edit Permissions">

    <Info>
      The Workday HRIS integration requires **all** of the permissions listed below to be enabled. 

      The Workday API will return an error response if any of these permissions are missing.
    </Info>

    Edit the Domain Security Policy Permissions in the Security Group. 

    To access this interface:
    1. Search for "Maintain Permissions for Security Group" in the search bar
    2. Select the Security Group you created in the previous step

    Enable the following **GET** operations for the HRIS API:

    - **Job Requisition Data**
    - **Job Information**
    - **Job Profile**
       - View
    - **Manage**
      - Location
      - Organization Integration
    - **Person Data**
        - Name
        - Citizenship Status
        - Date of Birth
        - Disabilities
        - Gender
        - Government IDs
        - ID Information
        - Marital Status
        - National ID Identification 
        - Personal Information
        - Personal Data
        - Personal Photo
      - Home Contact Information
      - Home Address
      - Home Email
      - Home Phone
      - Work Contact Information
      - Work Address
      - Work Email
      - Work Phone
    - **Worker Data**
      - All Positions
      - Compensation
      - Current Staffing Information
      - Employment Data
      - Organization Information
      - Public Worker Reports
      - **Time Off**
        - Time Off
        - Time Off Manager View
        - Time Off Balances
        - Time Off Balances Manager View
      - Workers
    - **View**
      - National Identifiers - All
  </Step>
</Steps>

## Approve the Security Policy Changes

<Steps>
  <Step title="Activate Security Policy Changes">
    1. In the Search bar, enter "Activate Pending Security Policy Changes"
    2. Review the summary of pending security policy changes
    3. Approve the changes to activate the new security policies
  </Step>
</Steps>

Use the Tenant and System User credentials you created in Steps 1 & 2 to establish the link with Workday.
