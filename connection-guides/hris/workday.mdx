## Finding your Workday Tenant

<Steps>
  <Step title="Log into Workday">
    Log into your Workday account. In the address bar at the top of your browser window, locate your tenant name. It appears immediately after `workday.com/`.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Workday Tenant" src="/images/workday/image1.png" />
    </Frame>
  </Step>
</Steps>

## Finding the Web Services Endpoint

<Steps>
  <Step title="Navigate to Public Web Services">
    Access the `Public Web Services` report.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Human Resources Web Service" src="/images/workday/image2.png" />
    </Frame>
  </Step>

  <Step title="Locate Human Resources">
    Find the Human Resources section and hover over it to access the menu. Click the three-dots menu, select `Web Service`, then click `View SDL`. Note that the page may take a moment to load completely.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="WSDL Service" src="/images/workday/image3.png" />
    </Frame>
  </Step>

  <Step title="Find the Service Location">
    Either search for `wsdl:service` or scroll to the bottom of the page. You should see the following:

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="WSDL Service" src="/images/workday/image4.png" />
    </Frame>

    Copy the portion of the location attribute that appears before `service`. Do not include the `https://` prefix. For example, in the tenant shown above, you would copy `wd2-impl-services1.workday.com/ccx`. Your tenant may have a different address (e.g., `wd5-services1.myworkday.com/ccx`).
  </Step>
</Steps>

## Set up an Integration System User

<Steps>
  <Step title="Access Integration System User Creation">
    In the Workday portal, use the Search field to locate "Create Integration System User".

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Create Integration System User" src="/images/workday/image5.png" />
    </Frame>
  </Step>

  <Step title="Select the Creation Task">
    Click on the "Create Integration System User" task.
  </Step>

  <Step title="Configure Account Details">
    In the Account Information section, enter your desired username and password.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Account Information" src="/images/workday/image6.png" />
    </Frame>
  </Step>

  <Step title="Confirm Creation">
    Click the OK button to create the Integration System User.
  </Step>
</Steps>

## Add the Integration System User to a Security Group

<Steps>
  <Step title="Initialize Security Group Creation">
    Use the Search field to find "Create Security Group" and select the corresponding task.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Create Security Group" src="/images/workday/image7.png" />
    </Frame>
  </Step>

  <Step title="Configure Security Group">
    On the creation page:
    1. Select "Integration System Security Group" from the Type of Tenanted Security Group dropdown menu
    2. Enter a descriptive name in the Name field

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Security Group Type" src="/images/workday/image8.png" />
    </Frame>
  </Step>

  <Step title="Save Security Group">
    Click the OK button to create the group.
  </Step>

  <Step title="Complete Group Configuration">
    On the "Edit Integration System Security Group (Unconstrained)" page, enter the same name you used when creating the Integration System User in the previous section. Click OK to save.
  </Step>
</Steps>

## Configure Domain Security Policy Permissions

<Steps>
  <Step title="Set Required Permissions">

    <Info>
      The Workday HRIS integration requires **all** of the permissions listed below to be enabled. 

      If any permissions are missing, the Workday API will return an error response.
    </Info>

    Configure the Domain Security Policy Permissions in the Security Group:

    1. Search for "Maintain Permissions for Security Group" in the search bar
    2. Select the Security Group you created earlier
    3. Enable the **GET** operations for these required HRIS API permissions:

    - **Job Requisition Data**
    - **Job Information**
    - **Job Profile**
       - View
    - **Manage**
      - Location
      - Organization Integration
    - **Person Data**
        - Name
        - Citizenship Status
        - Date of Birth
        - Disabilities
        - Gender
        - Government IDs
        - ID Information
        - Marital Status
        - National ID Identification 
        - Personal Information
        - Personal Data
        - Personal Photo
      - Home Contact Information
      - Home Address
      - Home Email
      - Home Phone
      - Work Contact Information
      - Work Address
      - Work Email
      - Work Phone
    - **Worker Data**
      - All Positions
      - Compensation
      - Current Staffing Information
      - Employment Data
      - Organization Information
      - Public Worker Reports
      - **Time Off**
        - Time Off
        - Time Off Manager View
        - Time Off Balances
        - Time Off Balances Manager View
      - Workers
    - **View**
      - National Identifiers - All
  </Step>
</Steps>

## Approve the Security Policy Changes

<Steps>
  <Step title="Activate Security Changes">
    1. Search for "Activate Pending Security Policy Changes"
    2. Review the summary of pending security policy changes
    3. Approve the changes to activate the new security policies
  </Step>
</Steps>
