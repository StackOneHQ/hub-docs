If you've been directed to StackOne to integrate with Salesforce, follow these steps to configure a successful integration.

## Generating Your API Credentials

<Steps>
  <Step title="Sign in to Salesforce">
    Sign in to Salesforce Developer and click on the Setup ⚙️ icon in the top-right menu.
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Setup icon" src="/images/salesforce/image1.png" />
    </Frame>
  </Step>

  <Step title="Navigate to App Manager">
    In the Platform Tools section of the left navigation, select Apps > App Manager. Then, click the New Connected App button in the top-right corner.
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="App Manager" src="/images/salesforce/image2.png" />
    </Frame>
  </Step>

  <Step title="Enter Application Information">
    In the New Connected App Basic Information section, enter your application-specific details:
    <ul>
      <li>Connected App Name</li>
      <li>API Name</li>
      <li>Contact Email</li>
    </ul>
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Application Information" src="/images/salesforce/image3.png" />
    </Frame>
  </Step>

  <Step title="Enable OAuth Settings">
    Check the Enable OAuth Settings box to reveal additional OAuth configuration options. In the Callback URL field, enter the StackOne OAuth2 URL:
    <code>https://api.stackone.com/connect/oauth2/salesforce/callback</code>
  </Step>

  <Step title="Select OAuth Scopes">
    Under Selected OAuth Scopes, select these required permissions:
    <ul>
      <li>Manage user data via APIs</li>
      <li>Perform requests at any time</li>
    </ul>
    Leave the default selections enabled for:
    - Require Secret for Web Server Flow
    - Require Secret for Refresh Token Flow
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="OAuth Scopes" src="/images/salesforce/image4.png" />
    </Frame>

    <Note>
      You do not need to enable any additional options in the OAuth section
    </Note>
  </Step>

  <Step title="Save the Settings">
    Click Save to proceed to the connected app page. Use the Manage Consumer Details button to view your Client ID and Client Secret. You may need to verify your account credentials at this step. Copy and securely store these credentials - you'll need them to configure your StackOne Hub integration.
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Client ID and Client Secret" src="/images/salesforce/image5.png" />
    </Frame>
  </Step>
</Steps>

## Connecting with StackOne Hub

<Steps>
  <Step title="Create a New Integration">
    Start a new integration by selecting Salesforce as your CRM provider.
  </Step>

  <Step title="Enter Credentials">
    Enter your Client ID, Client Secret, and your Salesforce URL in the following format:
    <code>https://your-domain.my.salesforce.com</code>
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Salesforce URL" src="/images/salesforce/image6.png" />
    </Frame>
  </Step>

  <Step title="Connect">
    Click the Connect button. A Salesforce tab will open for you to review and approve the connection.
  </Step>
</Steps>

## Troubleshooting

### Error: The REST API is not enabled for this Organization

This error occurs when your Salesforce account doesn't have API access enabled. API access varies by Salesforce edition (see [Salesforce Editions](https://help.salesforce.com)). We recommend using a Developer Edition account, which includes API access by default.

### Error: invalid_client_id when Linking the Account

If you receive this error:
<code>error=invalid_client_id&error_description=client%20identifier%20invalid</code>

This indicates that your Salesforce credentials are still being processed. While Salesforce states this can take up to 10 minutes, we've observed processing times of up to 20 minutes. Please wait and try again.

### Error: redirect_uri must match configuration

If you receive this error:
<code>error=redirect_uri_mismatch&error_description=redirect_uri%20must%20match%20configuration</code>

This means your Salesforce Connected App's redirect URI is incorrect. Ensure it matches exactly:
<code>https://api.stackone.com/connect/oauth2/salesforce/callback</code>
as specified in the OAuth Settings section above.
