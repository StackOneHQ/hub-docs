## Create Integrations App

<Steps>
  <Step title="Log into 360 Learning">
    Log into your 360 Learning account.
  </Step>

  <Step title="Navigate to Parent Group">
    Select Root Parent Group under `All Groups`.

    Click on `Settings` in the top right corner.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Dropdown Menu" src="/images/360learning/image1.png" />
    </Frame>
  </Step>

  <Step title="Configure API Key">
    From the left settings menu, select `API`.
    
    Locate the `Company ID`, in the Authentication section.
    
    Click `Add API Key` to create a new API key.
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Dropdown Menu" src="/images/360learning/image2.png" />
    </Frame>
    
    Enter `API Key` and click `Create`.
    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Dropdown Menu" src="/images/360learning/image3.png" />
    </Frame>

    Record the `Company ID` and `API Key` for use in the next steps.
  </Step>

  <Step title="Find Group ID">
    Select the desired group to find the `Group Id` in the address bar.

    E.g. If your url is `https://app.360learning.com/group/67c948a88723b46e710295bd/content/all`, the Group Id is `67c948a88723b46e710295bd`.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Dropdown Menu" src="/images/360learning/image4.png" />
    </Frame>
  </Step>

  <Step title="Find User ID">
    Navigate to `Group` -> `Users`.

    Open the User Profile to find the `User Id` in the address bar.

    E.g. If your url is `https://app.360learning.com/group/67c948a88723b46e710295bd/stats/users/current/users/67d14b2671c8825331af35f1`, the User Id is `67d14b2671c8825331af35f1`.

    <Frame>
      <img className='rounded-md' style={{ margin: '0 auto', border: '1px solid #efefef' }} alt="Dropdown Menu" src="/images/360learning/image5.png" />
    </Frame>
  </Step>
</Steps>

## SSO Configuration

Proper SSO (Single Sign-On) configuration is essential for the 360 Learning integration to work correctly. The configuration ensures that users are properly identified and matched between your platform and 360 Learning.

<Warning>
  Before configuring SSO, check the authentication approach with your content provider to ensure compatibility and proper user mapping.
</Warning>

### Configuration Options

The SSO configuration should use the following approach:

#### Custom Field Configuration
<Steps>
  <Step title="Set Custom Field">
    Configure a custom field called `lms_user_id` and set it to the **Email** field value.
  </Step>
  
  <Step title="Verify Custom Field">
    Ensure the `lms_user_id` custom field is properly populated and accessible through the SSO configuration.
  </Step>
</Steps>

<Warning>
  **Important:** Misconfigured SSO settings can result in users being unable to access their learning content or receiving incorrect learning records. Always verify the field mapping is accurate.
</Warning>

### Customer Responsibility

- Ensure the Email field is correctly populated in your user management system
- Verify that the SSO configuration matches the authentication method used by 360 Learning
- Test the SSO flow with a small group of users before full deployment
